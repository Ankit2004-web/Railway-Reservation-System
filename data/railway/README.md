# Railway Master Data — Data Directory

## Classification

| Category | Description | Location |
|----------|-------------|----------|
| **A — Master** | Stations, trains, routes, schedules (static) | `processed/*.csv` |
| **B — Dynamic** | Availability, fares simulation, bookings | Generated by application |
| **C — Real-time** | Live location, delays | Not included — future adapters |

## IMPORTANT

- Files in `processed/` included here are **DEVELOPMENT / TEST DATA**
- They are **NOT** official Indian Railways / IRCTC data
- Do **not** commit large government datasets to git — use `raw/` locally

## Folder layout

```
data/railway/
  raw/          ← place downloaded open-government CSV/JSON here (gitignored if large)
  processed/    ← normalized CSV ready for import
  archive/      ← previous import snapshots
  RailwayDataImportReport.json  ← generated after import
```

## Import command

```bash
npm run db:sync
npm run import:railway
```

Or with custom directory:

```bash
node database/import/run-import.js --dir path/to/csv/folder
```

## Legitimate data sources (manual)

When you obtain authorized open datasets, record provenance in `DataImportSources` via import metadata:

- [data.gov.in](https://data.gov.in) — Government of India Open Government Data
- Ministry of Railways public datasets (verify license before import)

**Do not scrape IRCTC or private endpoints.**
